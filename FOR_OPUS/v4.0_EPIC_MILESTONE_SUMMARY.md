### **The Philosophy of Version Numbers**

Version numbers aren't just technical markersâ€”they're **promises to users** about what coheres and what doesn't. They answer the question: *"What moves together?"*

**Two versioning philosophies:**

1. **Microservice Independence** - Each component versions separately
   - **Good for:** Large distributed systems, stable APIs, external dependencies
   - **Signal:** "These pieces can evolve independently"
   - **Example:** Linux kernel 6.x using GCC 13.x

2. **Monolithic Cohesion** - Everything ships together under one version
   - **Good for:** Integrated applications, tightly coupled systems, user-facing products
   - **Signal:** "This is one coherent whole"
   - **Example:** iOS 17.0 (all frameworks ship together)
