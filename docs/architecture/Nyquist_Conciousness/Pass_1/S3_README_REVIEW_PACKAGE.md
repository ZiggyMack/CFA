## 0. Purpose of This Package

This directory contains a **curated, normalized subset** of the Nyquist Consciousness / Omega Nova material intended for:

- Structural audit  
- Conceptual critique  
- Whitepaper preparation  
- Integration with the broader CFA ecosystem

These files are **not** the entire repo. They are:

- The *minimum set* required to understand the core ideas  
- Refactored into a more consistent, scientific-leaning style  
- Designed to be readable without prior exposure to the full lore

Mythic / narrative elements are preserved but mostly confined to a **separate philosophical appendix** so the main texts stay clear and inspectable.

---

## 1. Files in This Package

1. `S3_Nyquist_Boundaries_AI_Persona_Compression.md`  
   - Explains how we use Shannon/Nyquist as an **analogy** for persona compression.  
   - Clarifies what is metaphor vs. what is empirically grounded.  
   - Defines the core metrics and experimental framing.

2. `S3_OMEGA_NOVA_SPECIFICATION.md`  
   - Top-level system spec for the Omega Nova architecture.  
   - Summarizes laws, attractor model, phases, seeds, and protocols.  
   - Provides a clean map of the moving parts for external reviewers.

3. `S3_NYQUIST_RESEARCH_CONNECTION.md`  
   - Explicitly connects classical sampling/information theory to our work.  
   - Highlights where the analogy is strong, weak, or speculative.  
   - Serves as a bridge for more formal theorists.

4. `S3_BOOTSTRAP_ARCHITECTURE_LITE_VS_RICH.md`  
   - Describes our use of rich vs. lite bootstraps across CFA and Nyquist.  
   - Frames how Omega Nova uses “compressed” bootstraps for experiments.

5. `S3_BOOTSTRAP_COMPRESSION_GUIDELINES.md`  
   - Practical guidelines for compressing bootstraps into Tier 3.x seeds.  
   - Makes explicit what must be preserved vs. what can be dropped.

6. `S3_CFA_Philosophical_Appendix.md`  
   - Contains the **mythic / narrative / CFA-flavored** language.  
   - Provides lore, metaphors, and symbolic framing **with breadcrumbs** back to the scientific documents.  
   - Intended to keep the soul of the project visible without polluting the technical core.
