### 5.2 Canonical YAML File Structure

**File Purpose:**
- Machine-readable single source of truth for worldview scores
- Used by Console, validation tools, and empirical verification

**Top-Level Blocks:**

1. **profile:** Metadata (name, version, status, axiom declaration)
2. **axioms:** List of unprovable assumptions with descriptions
3. **debts:** List of unresolved explanatory burdens with descriptions
4. **levers:** 6 YPA scoring dimensions (0-10 scale)
5. **behavioral_flags:** Boolean flags (admits_limits, etc.)
6. **calculated:** Pre-calculated values for verification

**Calculated Block (Verification Aid):**
- Stores pre-calculated BFI, YPA, and preset variations
- Purpose: Enables quick sanity checks without recalculation
- Example:
  ```yaml
  calculated:
    axiom_count: 7           # Count of axioms list
    debt_count: 4            # Count of debts list
    bfi_standard: 11.0       # (7 + 4) * 1.0
    lever_sum: 48.0          # Sum of all 6 levers
    ypa_standard: 4.36       # 48.0 / 11.0
    skeptic_bfi: 13.2        # (7 + 4) * 1.2
    skeptic_ypa: 3.64        # 48.0 / 13.2
    diplomat_bfi: 9.9        # (7 + 4) * 0.9
    diplomat_ypa: 4.85       # 48.0 / 9.9
    zealot_bfi: 7.7          # (7 + 4) * 0.7
    zealot_ypa: 6.23         # 48.0 / 7.7
  ```

**File Header (Comments):**
```yaml
# Classical Theism - Canonical Score Data
# Purpose: Machine-readable YPA scores for validation and calculation
# Source: Extracted from CLASSICAL_THEISM.md (v0.3.0)
# Usage: ypa_validator.py, empirical verification, Grok validation
# Last Updated: 2025-11-15
```
